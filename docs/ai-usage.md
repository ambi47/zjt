# AI 工具使用说明（课程作业说明）

本项目开发过程中使用了 AI 辅助工具提升效率，主要用于：

## 使用场景

1. 需求拆解与实现路线
   - 将“数据库配置/连接/表设计/后台界面/二级页面”等需求拆成可执行任务，并给出可落地的实现方案。

2. Bug 定位与修复
   - 通过错误日志与页面现象定位原因（例如脚本资源加载、URL 重定向处理、头像更换逻辑冲突等），并提出修复代码。

3. 代码实现与重构
   - 在现有 Express + JWT 架构下新增数据库模块、后台管理 API、资源详情弹窗等功能，同时尽量沿用项目原有结构与代码风格。

4. 文档整理
   - 输出数据库表设计说明、演示说明，方便课堂汇报与作业提交。

## 工具组合

- Trae IDE 内置 AI 助手：用于在本地项目中直接定位文件、生成修改补丁、运行命令验证结果。
- 大语言模型（对话式）：用于提出实现建议、生成可运行的代码框架、辅助排查错误。

## 结果验证

- 通过本地启动服务并访问页面/接口验证功能可用。
- 数据库文件在 `server/data.sqlite`，后端启动后会自动初始化表结构并填充基础数据。
